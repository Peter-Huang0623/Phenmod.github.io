使用 ClangFormat 插件格式化Objective-C代码

Nov 17th, 2015 8:26 pm

插件地址：https://github.com/travisjeffery/ClangFormat-Xcode

这个是我们自己定义的格式配置文件.format

这个是Parse的配置文件：https://github.com/ParsePlatform/xctoolchain/blob/master/.clang-format

每次提交前，必须格式化代码。 用了这个之后，基本上就不用开发者做排版的事情了，哗啦啦写完代码，一键格式化，人生舒爽了很多。

关于空行： 配置文件里我们现在允许最多连续两行空行，正在考虑只允许一行。 确实有分段视觉需求的，使用pragma mark进行分割，如下： // =================================================================================================

pragma mark - <#代码分块注释#>

// ================================================================================================= 用它定义个code snippet，来个快捷键(我的是pm)，省心啊。

关于每行字符限制： 如果你把上面的pragma段拷贝到xcode，会发现上面那一串等号会终止在第100列。没错，我们现在每行字符数限制就是100。 其实我很想定义到120，但是在某些同学的macbook air的小屏幕上不太够显示。
